# 質問
1. rbenvとは、Rubyのバージョン管理ツールとしてどのような機能を提供しますか？
2. rbenvの基本的な使い方（インストール、バージョン指定、切り替えなど）を教えてください。
3. rbenvを使うことのメリットとデメリットは何ですか？

# 回答
**1. rbenvとは、Rubyのバージョン管理ツールとしてどのような機能を提供しますか？**
rbenvはRubyのバージョン管理ツールであり、同じマシン上で複数のRubyのバージョンを切り替えて使用することができます。

**2. rbenvの基本的な使い方（インストール、バージョン指定、切り替えなど）を教えてください。**
rbenvを使用するための手順は以下の通りです。

**① インストール（準備）**
Homebrewを使ってmacOSにrbenvとruby-buildをインストールします。
```bash
brew install rbenv ruby-build
```

**② シェル設定の追加（初期化）**
rbenvが使えるように、シェル（~/.zshrc）の設定ファイルに以下のコマンドを追加し、シェルのPATHにrbenvの実行ファイルがあるディレクトリを追加し、rbenvを初期化します。
```bash
echo 'export PATH="$HOME/.rbenv/bin:$PATH"' >> ~/.zshrc
echo 'eval "$(rbenv init -)"' >> ~/.zshrc
```
その後、`source ~/.zshrc`を実行するか、ターミナルを再起動します。

**③ バージョンのインストールと確認**
1.  **一覧取得:** インストール可能なバージョン一覧を取得します。
    ```bash
    rbenv install --list
    ```
2.  **インストール:** 利用したいバージョンを指定してインストールします。（例：3.3.3）
    ```bash
    rbenv install 3.3.3
    ```
3.  **確認:** インストールされているバージョンを確認します。
    ```bash
    rbenv versions
    ```
4.  **表示確認:** 現在のRubyのバージョンを表示します。
    ```bash
    ruby -v
    ```

**④ バージョンの切り替え**
システム全体のデフォルトのRubyバージョンを切り替えます。
```bash
rbenv global 3.3.3
```
切り替え後、`ruby -v`でバージョンを確認します。

**3. rbenvを使うことのメリットとデメリットは何ですか？**
*   **メリット:**
    *   同じマシン上で複数のRubyのバージョンを切り替えることができる点です。
*   **デメリット:**
    *   提供された文脈中には、rbenvを使用することのデメリットについての記述はありません。
